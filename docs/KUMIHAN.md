# 組版案内（現代版） ── 注記テキストのまとめ方、Unicode中間表現への変換法

本書は、青空文庫の「[組版案内](http://kumihan.aozora.gr.jp/)」（2010–2012、
txt2xhtml＝t2hs.rb の公式ホーム）を、本リポジトリの実装に合わせて書き改めた
現代版である。本家は「原則としてあるがままの状態で」凍結されており、
Ruby 1.9・Shift_JIS・外字グリフ画像を前提としている。本書はその**思想と
書式を引き継ぎ**、前提だけを2026年の環境（Python・Unicode・実文字外字）に
置き換える。

注記の**書式そのもの**の正本は、今も本家の「注記一覧」である
（ミラー: `raw.githubusercontent.com/aozorabunko/aozorabunko/master/annotation/`）。
本書はそれを書き写さない。ここにあるのは「その書式を、この道具箱が
どう読み、何に変換するか」である。

## 1. テキスト版と、その変換先

テキスト版は本家の定めどおり: JIS X 0201（半角カナ除く）＋JIS X 0208 の文字、
Shift_JIS、改行 CR+LF、注記は `《》` と `［＃…］`。本ツール群はこれを
そのまま読む（UTF-8 のファイルも自動判別で読める）。

本家の変換先は XHTML 版ひとつだった。本書の体系では、変換の一次成果物は
**Unicode 中間表現（Document）**であり、そこから各形式を生成する:

```
注記テキスト（Shift_JIS/UTF-8）
   │  pybunko.parse … 外字・アクセントは実文字に解決
   ▼
Document（Unicode中間表現・JSONで保存可能）
   ├→ 公式XHTML互換      pybunko.to_official_html（本家t2hsの再現・検証用）
   ├→ 書架SQLite・読者アプリ「文庫」（Flutter: 縦書き・ページめくり・TTS）
   ├→ EPUB／washi-md縦書きHTML・PDF／朗読パック（Opus＋段落タイミング）
   └→ プレーンテキスト・読み上げ用テキスト
```

本家 FAQ が言う「aozora.css が2階層上」「図書カードリンクが動かない」の類は、
公開サーバのディレクトリ構造をハードコードした t2hs の癖である。互換出力は検証のため癖ごと再現する。読書用の出力は Document 側から
自己完結に作る。

## 2. 注記一覧と対応状況

本家「注記一覧」の目次に沿って、`pybunko.parse` の対応を示す。
未対応の注記は**黙って消えるのではなく** `parse(text, unknown_notes=リスト)`
で報告され、工房の検査タブに一覧表示される。

| 分類 | 注記 | 対応 | Document上の表現 |
|---|---|---|---|
| レイアウト | 字下げ（1行・ブロック・凹凸） | ○ | `indent`（入れ子対応） |
| | 地付き・地から○字上げ | ○ | `align` / `align_offset` |
| | 字詰め | ○ | `jizume` |
| | 改丁・改ページ・改段 | ○ | `pb`（マーカー段落。読者アプリは区切りとして描画） |
| | ページの左右中央 | ○ | `pb: center`（マーカー） |
| 見出し | 大・中・小 × 通常・同行・窓（前置/後置/ブロック） | ○ | `heading_level` 2/3/4＋`heading_type` |
| | 目次 | ×（アプリが見出しから自動生成） | ― |
| 外字 | 第3・第4水準の面区点、`U+xxxx` | ○ | **実文字**（jis2ucs 11,233字） |
| | 面区点の無い外字 | △ | 記述を保持（推測しない） |
| | アクセント分解 `〔…〕` | ○ | 実文字（é 等） |
| 訓点 | 返り点・訓点送り仮名 | × | ― |
| 強調 | 傍点9種・傍線5種 | ○ | `decorations` |
| | 太字・斜体 | ○ | `decorations` |
| 画像 | 挿絵 `（fig….png…）入る` | ○ | `image`（ミラーURL参照） |
| | キャプション | × | ― |
| その他 | 訂正と「ママ」・「…」の注記・左注記 | ○ | `notes`（本文は訂正後の形のまま、正誤を構造で保持） |
| | ルビ・`｜` | ○ | `segments`（本文とルビの対） |
| | 縦中横・割り注 | × | ― |
| | 上付き・下付き小文字 | ○ | `decorations`（sub/sup） |
| | 罫囲み・横組み・文字サイズ | × | ― |

×の注記は本家 t2hs が正しく変換する（新着作品でも文字サイズ注記が動いている
のを確認した）。互換出力の完全一致を目指すときは、この表の×を埋めることが
「永久作業」の続きになる（手順は MANUAL.md §注記の増やし方）。

どの×から埋めるかは勘でなく**一括点検**（`python -m pybunko.census`／工房の
検査タブ）で決める。2026-07-12 の400作品調査では、出現作品数の上位は
訂正49・ママ30・地から字上げ後置形23・改ページ14作品——この4種を対応済みに
した。次点は文字サイズ（17作品）・訓点（14作品）・横組み（13作品）。

## 3. 変換してみる

本家はサイト上の変換フォームを提供した。本書の体系では**青空工房**
（`app/pykobo/aozora_kobo.py`、Flet製）がそれにあたる:

- **検査タブ** …… 作品を検索して変換し、未対応注記・未解決外字（〓）・
  統計・プレビューを確認する（本家フォームの「通るかどうか確かめる」に相当）
- **検証タブ** …… 互換出力を、公開されている本物のXHTMLと突き合わせて
  diff 表示する（類似度%）。互換性の正は常に**実際の公開ファイル**に置く
- **入力・校正タブ** …… 作業マニュアル準拠のチェックと下書き
  （組版案内の範囲外だが、同じ土台の上にある）

起動: `KOBO_PORT=8789 python aozora_kobo.py`（スマフォからも同一LANで使える）

## 4. 自分のPCで変換

本家: `$ ruby1.9 t2hs.rb input.txt output.html`（要 Ruby 1.9、Shift_JIS のみ）

本書の体系（Python 3.10+、追加依存なし）:

```bash
# 注記テキスト → Unicode JSON（一次成果物）
python -m pybunko 入力.txt -o 作品.json

# 注記テキスト → 公式XHTML互換（t2hs.rb 相当）
python -c "from pybunko import to_official_html, read_text; \
           print(to_official_html(read_text('入力.txt')))" > 出力.html

# 機械チェック（チェッカー君・校閲君・点検グループ正規表現の相当品）
python -m pybunko.kosei 入力.txt

# 印刷用PDF（washi-md 組版: 縦書き・ルビ・禁則。要 [washi] エクストラ＋Chrome）
python -c "from pybunko import parse, read_text; from pybunko.formats import to_pdf; \
           to_pdf(parse(read_text('入力.txt')), '出力.pdf', vertical=True)"
```

印刷は本家に無かった出力形式である。組版の実体は aiseed-dev/washi-md
（MIT・青空の傍点・ルビ記法を上流還元済み）に委譲しており、原稿用紙
（genko=True）・テーマ切替も同じ入口から使える。工房の検査タブにも
「印刷用PDF」「組版HTML」ボタンがある。

外字グリフ画像は不要になった。外字は実文字で埋め込まれ、表示は
JIS X 0213 対応フォント1つ（読者アプリ同梱の IPAex明朝、または
`pybunko.fonts.build_gaiji_font()` で作るサブセットWOFF2）で足りる。

## 5. 何に使うの？

本家の三つの用途は、そのまま現代版に写せる:

1. **作業中のチェック** …… 入力・校正中のテキストが変換を通るか、
   注記付けに誤りがないかを自分で確かめる（工房の検査・校正タブ）
2. **電子出版** …… 注記体系とその変換器を使って自分の出版物を作る。
   本家 t2hs は Public Domain、本リポジトリのコードは AGPL-3.0
3. **表示ソフトの開発** …… 仕様のよりどころ。ただし本書の体系では、
   表示ソフトが読むべきは XHTML ではなく **Document JSON**
   （スキーマは MANUAL.md）。読者アプリ「文庫」がその参照実装である

## 6. 配布条件と出典

## 7. 本家との既知差分（2026-07-12 時点）

新着「人工衛星の発射地」（2026-07-11公開）との突き合わせで確認:

1. 文字サイズ注記 → 本家は `<span class="sho1" …>` を出す。互換出力は未対応
2. 末尾の定型文 → 本家の新しいファイルは「青空文庫」にアンカーを張らない
   （古いファイルは張る。互換出力は古い形を再現する）
3. 外字 → 本家はグリフ画像（`class="gaiji"`）、本書の体系は既定で実文字。
   画像互換モードも `to_official_html(gaiji='image')` に温存
